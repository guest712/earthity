# Earthity — roadmap

Краткая карта фаз. **Рабочие задачи** — в `EARTHTY-BACKLOG.md`. **Документация** — `docs/README.md` (продукт, питч, бизнес, архитектура). **Идеи на годы** — в `frankintsugi.ini`. **Доверие UGC (фото, модерация)** — в `.E_plans_tasks/validation_check.md`.

**Сейчас:** фаза **0** (порядок в репо + документация), затем фаза **1** (стабильный тест в одном городе).

---

## Фаза 0 — Порядок (≈ неделя)

| Трек | Цель |
|------|------|
| Репо | Закоммитить незакоммиченное (epic reward, placement); решение по GLB в git |
| Документы | `docs/*` готовы; бэклог синхронизирован; миграции 002–004 применены в Supabase |
| Код | P3: удалить мёртвый REST-auth; `.env.example`; краткий `docs/SUPABASE.md` |

**Готово, когда:** `git status` чистый (кроме осознанно игнорируемых GLB), новый разработчик понимает Supabase за 10 минут.

---

## Фаза 1 — Стабильный онлайн-цикл (≈ 1–2 недели)

**Цель:** отдать сборку друзьям в **одном городе** (центр epic сейчас: Berlin 52.52 / 13.405).

| Трек | Задачи |
|------|--------|
| Backend | P2: dev/prod Supabase, confirm email (prod), сверка RLS `saves` + `cleanup_spots` |
| Frontend | Карта: спокойный вид (`customMapStyle`, HUD); полировка меток |
| Безопасность | Без ИИ; лимиты меток (3 active, 7 дней TTL) уже в коде |
| Проверка | 2+ аккаунта: метка → epic → уборка → награда → сейв → logout/login |

---

## Фаза 2 — Доверие к UGC (после первого внешнего теста)

По `.E_plans_tasks/validation_check.md`, по шагам:

1. Supabase Storage + RLS, `proof_url` при «убрал»
2. Камера-only, сжатие на клиенте
3. Ручная модерация (`pending` → `open`) или report + залог кармы
4. Связать награду / epic с наличием proof (опционально)

**Не в фазе 2:** ИИ «на фото это мусор».

---

## Фаза 3 — Игровые петли (по одной за итерацию)

| Приоритет | Петля | Задел в коде |
|-----------|--------|----------------|
| 1 | Стадии существ (рост от ухода) | `MapARScene`, `careDiary`, `stages` в types |
| 2 | Daily / weekly (календарь) | `dailyQuests`, частично есть |
| 3 | Крафт / переработка мусора | вкладка craft, ресурсы |
| 4 | POI на карте (не cleanup) | отдельно от `cleanup_spots` |
| 5 | Пуши по стрику / уходу | `expo-notifications` |

Каждая петля: UI → локальный сейв → при необходимости облако.

## Фаза 4 — Социал и масштаб (месяцы)

## Правила работы

1. Одна фаза + одна петля за спринт — не смешивать с frankintsugi целиком.
2. Коммит перед большой задачей в Cursor; эксперименты — ветка `experiment/…`.
3. Новая миграция Supabase → файл в `supabase/migrations/` + Run в SQL Editor.
4. В чат агенту: «работаем по EARTHTY-BACKLOG, фаза N».
